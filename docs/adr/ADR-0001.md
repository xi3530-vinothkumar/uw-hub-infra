# ADR-0001: AI perceives & explains; deterministic rules decide

**Status:** Accepted

## Context
The tool must be defensible and auditable (regulators, loss reviews, senior UWs). LLM
outputs are non-deterministic and can hallucinate.

## Decision
AI is used only for perception (text extraction, photo vision) and explanation (narrative).
The composite risk score and the Accept/Refer/Decline band are produced by a deterministic,
rules-based engine in Java. The LLM never produces the number or the recommendation.

## Consequences
+ Reproducible, explainable, testable decisions; audit trail written by code.
+ LLM failure degrades explanation only, not the decision.
- We maintain an explicit scoring model (weights/thresholds) rather than "letting the model decide".

## Alternatives considered
- LLM-produced score/recommendation: rejected — not reproducible or defensible.
