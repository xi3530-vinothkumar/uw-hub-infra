# ADR-0003: Local Ollama models, model-agnostic via adapter

**Status:** Accepted

## Context
Requirement: cloud-portable, model-agnostic, open-source-leaning; only a Claude *subscription*
is available (no runtime API billing). Build with Claude Code, run on free/open models.

## Decision
Run open-weight models locally via Ollama (`llama3.1:8b` for text/narrative,
`qwen2.5vl:7b` for vision; `llava:7b` fallback). All access goes through one
`LLMAdapter`, so the provider is swappable (Claude/GPT/Gemini) via config only.

**Amended 2026-07-03:** vision model changed `llama3.2-vision:11b` → `qwen2.5vl:7b`.
Current Ollama dropped the `mllama` architecture, so llama3.2-vision no longer loads;
qwen2.5vl:7b was chosen over llava:7b (better fine-grained damage perception and
structured-JSON discipline at similar size). Config-only change (`MODEL_VISION`),
per this ADR's adapter design.

## Consequences
+ Zero runtime API cost; private; satisfies open-source + model-agnostic goals.
+ Provider swap needs no business-logic change.
- Local models are weaker (esp. vision) and one-model-at-a-time (drives ADR-0004, ADR-0010).

## Alternatives considered
- Hosted commercial API at runtime: no API billing available; less "open-source".
- LiteLLM gateway: viable; deferred — a single adapter suffices for the MVP.
