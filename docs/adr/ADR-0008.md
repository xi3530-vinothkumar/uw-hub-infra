# ADR-0008: Versioned decisions (never overwrite)

**Status:** Accepted

## Context
Underwriters may override COPE after seeing a decision, forcing a re-score. Auditability
requires seeing the original AI proposal vs. the post-override result.

## Decision
`decisions` is versioned: each scoring run inserts a new row (`version+1`, `is_current`
flip); prior versions are retained. Overriding COPE after a decision supersedes it and
returns the submission to `REVIEWED` for re-scoring. Approval/export act on the current version.

## Consequences
+ Full decision history; strong auditability.
- Slightly more query logic (select current version).

## Alternatives considered
- Single mutable decision row: rejected — destroys history.
