# ADR-0002: Two services — Java orchestrator + Python AI worker

**Status:** Accepted

## Context
Team is strongest in Java; the AI/LLM ecosystem is strongest in Python. Decisions must be
deterministic and auditable; AI calls are messy (JSON parsing, image handling, retries).

## Decision
Java/Spring Boot owns orchestration, the rules engine, persistence, enrichment, and all
decisions (system of record). Python/FastAPI is a stateless AI worker owning only Ollama
calls. They communicate via RabbitMQ.

## Consequences
+ Each language does what it is best at; clean boundary; Python is replaceable.
+ Decision authority centralized in the mature, testable Java layer.
- Two runtimes to build and run; a message contract to maintain.

## Alternatives considered
- Single Python service: weaker fit for the team + the deterministic/auditable core.
- Single Java service calling Ollama directly: loses Python's AI ergonomics.
